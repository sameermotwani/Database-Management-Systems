Sample Exercise

A. Stored Procedures

Develop a stored procedure that computes the diameter of a graph starting from a specified vertex. Your procedure should be named diameter. It takes two parameters. The first is an in parameter that specifies the starting vertex id. The second is an out parameter that returns the diameter (i.e., the maximum distance from the starting vertex to any other vertex). You will need a test program. Submit your test program as well as the code that creates the stored procedure.

B. Transaction Management and Concurrency

For each of the following schedules, determine whether each one is conflict-serializable, view-serializable and strict. If it is conflict-serializable show the equivalent serial schedule, and explain why it is equivalent. If it is not conflict-serializable, show why. If it is view-serializable show the equivalent serial schedule, and explain why it is equivalent. If it is not view-serializable, show why. If it is strict, explain why. If it is not strict, explain why not. 

  1. R1(A), R2(B), W2(C), W3(A), W3(C), C3, W1(A), C1, C2

  2. R1(A), R1(C), R2(B), W1(C), R4(A), R3(C), W2(B), W4(A), W3(C), W2(C), C3, W1(A), R4(B), C1, W2(C), C4, C2
