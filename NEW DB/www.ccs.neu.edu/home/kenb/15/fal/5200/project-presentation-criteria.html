Rubric for Team Project Implementation and Presentation

The following is a summary of the parts of the presentation and demo. It begins with a description of the parts of a project report. Only certain parts of the report affect the score. The rubrics for the implementation and presentation are then shown.

Report Format

SectionDescriptionCommentsWhat should be checked (if anything)Maximum score for this part
TitleName of the report It must be descriptive of the project that was completed but must also be succinct. Not required in the presentationNo score for this part
AuthorsThe authors and their email addresses List them in alphabetical order. Not required in the presentationNo score for this part
AbstractSuccinct summary of the report. It should never have citations or footnotes, and it should minimize the use of technical terminology. Not required in the presentationNo score for this part
IntroductionThe first section of the report. It serves to state the purpose and objectives of the project. Be careful to distinguish the goals that are achieved by the project, and the goals that motivate the project but were not actually achieved. Check that this matches the Project Proposal submitted by the team. See the Conclusion below. 20 points for Presentation
RequirementsThe requirements for the software being produced. The Use Case Description Ontology is designed for formally specifying requirements and can be used for this section. Check that the use cases of the Project Requirements submitted by the team were all fulfilled. See the Discussion section below. 20 points for Presentation and 30 points for Implementation
DesignThe software design and rationales for the design. The design can be specified using UML diagrams. The design rationales and decisions are written in the accompanying text. Check that the team has a database or a data source that conforms to the Project Design submitted by the team. 20 points for Presentation
ImplementationDescription of the implementation. The implementation itself is not part of the report, but the work artifacts must be submitted. This is where one specifies the programming language(s) used as well as other software and frameworks. The team should give a demonstration of their software. It does not have to show all of the use cases. One or two should be enough. 70 points for Implementation (40 points for the demonstration and 30 points for submission of the work artifacts).
DiscussionThis section explains the results of the project in detail. Examples of how the software is used are important. This section should establish that the requirements were met. See the note below. The presentation should describe at least one example. 20 points for Presentation
ConclusionSummarize the results of the project and the extent to which the goals (in the Introduction) were achieved. One can also mention further work suggested by this project. This was already covered in the Introduction section above. There would normally be no time to mention any further work. Scored as part of the Introduction score.
ReferencesThe bibliography Try to use a standard bibliographic format. The references must be included in the presentation. 20 points for Presentation

Note: The Discussion section is actually several sections, each showing one example in detail. Each section should have a title appropriate to the example in that section.

Presentation Rubric

Presentation section(s)What should be checked (if anything)Maximum score for this part
Introduction and Conclusion The presentation must briefly explain the objectives of the project that were actually achieved. 20 points
Requirements The presentation must discuss how the project use cases were fulfilled. 20 points
Design The presentation must describe the database or data source and explain how it fulfills their design. 20 points
Discussion The presentation should describe at least one example of the use of the project software. 20 points
References The references must be included in the presentation. 20 points

Implementation Rubric

Implementation section(s)What should be checked (if anything)Maximum score for this part
Requirements The project must fulfill its requirements. 30 points
Demonstration The project must be well demonstrated. 40 points
Implementation The team must submit its work artifacts. 30 points
